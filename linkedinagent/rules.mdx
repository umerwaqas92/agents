# LinkedIn Agent Rules

## ADB Screen Reading

```bash
adb shell uiautomator dump /sdcard/screen.xml && adb pull /sdcard/screen.xml /tmp/screen.xml
```

Parse XML with:
```python
python3 << 'PYEOF'
import xml.dom.minidom
dom = xml.dom.minidom.parse('/tmp/screen.xml')
nodes = dom.getElementsByTagName('node')
for n in nodes:
    text = n.getAttribute('text')
    desc = n.getAttribute('content-desc')
    bounds = n.getAttribute('bounds')
    if text or desc:
        print(f"text='{text}' desc='{desc}' bounds='{bounds}'")
PYEOF
```

## Tapping Elements

Extract bounds from XML (format: `[x1,y1][x2,y2]`), calculate center:
```
center_x = (x1 + x2) // 2
center_y = (y1 + y2) // 2
```

Then tap:
```bash
adb shell input tap <center_x> <center_y>
```

### Step 5: Type Post Content

**RECOMMENDED** - Word-by-word with keyevent for spaces (avoids ALL encoding issues):
```python
python3 << 'PYEOF'
import subprocess, time

lines = [
    "Your post heading",
    "",
    "Your post body paragraph one",
    "",
    "Your post body paragraph two"
]

for i, line in enumerate(lines):
    if line.strip():
        for word in line.split():
            subprocess.run(['adb', 'shell', 'input', 'text', word], check=True)
            subprocess.run(['adb', 'shell', 'input', 'keyevent', '62'], check=True)  # space
            time.sleep(0.12)
        subprocess.run(['adb', 'shell', 'input', 'keyevent', '67'], check=True)  # backspace
    if i < len(lines) - 1:
        subprocess.run(['adb', 'shell', 'input', 'keyevent', '66'], check=True)  # Enter
        time.sleep(0.3)
PYEOF
```

## Commenting on a Post

**NOTE**: LinkedIn's comment input is a Compose view. `adb input text` may not work reliably. The send button only appears after text is successfully entered.

### Prerequisites
Ensure `uiautomator dump` works. If it returns `null root node`, check for crashed accessibility services:
```bash
adb shell dumpsys accessibility | grep "Crashed services"
```
If a service is crashed, reset:
```bash
adb shell settings put secure enabled_accessibility_services ""
```
Then restore after:
```bash
adb shell settings put secure enabled_accessibility_services "<original_value>"
```

### Steps

1. **Find posts** in the feed by dumping the XML and searching for Comment buttons:
```bash
adb shell uiautomator dump /sdcard/screen.xml && adb pull /sdcard/screen.xml /tmp/screen.xml
python3 << 'PYEOF'
import xml.dom.minidom
dom = xml.dom.minidom.parse('/tmp/screen.xml')
nodes = dom.getElementsByTagName('node')
for n in nodes:
    text = n.getAttribute('text')
    desc = n.getAttribute('content-desc')
    bounds = n.getAttribute('bounds')
    if text or desc:
        print(f"text='{text}' desc='{desc}' bounds='{bounds}'")
PYEOF
```

2. **Tap the "Comment" button** on the desired post (calculate center from bounds):
```bash
adb shell input tap 264 771  # Example: bounds=[216,723][312,819]
sleep 2
```

3. **Tap the comment input field** (look for "Add a comment..." in XML):
```bash
adb shell input tap 670 1380  # Example: bounds=[219,1349][1121,1412]
sleep 1
```

4. **Type comment** - split by words, type each word literally with delay, use keyevent 62 for space (avoids ALL hex ambiguity issues with `%s`):
```python
python3 << 'PYEOF'
import subprocess, time
comment = "Great post thanks for sharing"
for word in comment.split():
    subprocess.run(['adb', 'shell', 'input', 'text', word], check=True)
    subprocess.run(['adb', 'shell', 'input', 'keyevent', '62'], check=True)  # 62 = space
    time.sleep(0.1)  # Prevent dropped characters
# Remove trailing space with backspace
subprocess.run(['adb', 'shell', 'input', 'keyevent', '67'], check=True)  # 67 = backspace
PYEOF
sleep 1
```

5. **Tap send** (blue "Comment" button at bottom of sheet):
```bash
adb shell input tap 1021 1518  # Example: bounds=[895,1484][1148,1552]
sleep 2
```

6. **Verify**: Look for your comment text in the XML output with "now" timestamp:
```bash
adb shell uiautomator dump /sdcard/screen.xml && adb pull /sdcard/screen.xml /tmp/screen.xml
python3 -c "import xml.dom.minidom; dom=xml.dom.minidom.parse('/tmp/screen.xml'); [print(n.getAttribute('text')) for n in dom.getElementsByTagName('node') if n.getAttribute('text')]"
```

## Common Pitfalls

1. **URL-encoded text appears literally**: `adb shell input text` already URL-encodes. Don't pre-encode with `%20`, `%27`, etc. Pass raw text or use `%s` for spaces only.

2. **Special characters**: Characters like `'`, `"`, `$`, `#`, `?`, `.`, `,`, `-` get mangled. Either:
   - Strip them entirely (keep alphanumeric + spaces)
   - Use keyevent codes for each character (slow but accurate)
   - Use clipboard paste if available

3. **Shell escaping**: When using heredocs (`<< 'HEREDOC'`), single quotes around delimiter prevent variable expansion. Always use this for post content.

4. **Timing**: Always `sleep 1-3` after taps to let the UI update before dumping XML.

5. **Post button disabled**: The Post button (`enabled="false"`) becomes `enabled="true"` only when text is entered. Check this before tapping.

6. **Comment send button**: LinkedIn's comment input may not accept `adb input text` reliably. The send button (right side of comment bar) only appears when text is detected. If text doesn't appear, the send button won't show.

7. **Feed caching**: LinkedIn may show the same posts repeatedly. Use aggressive swipes (`input swipe 610 2200 610 200 1000`) to load new content.

8. **Crashed accessibility services block uiautomator**: If `uiautomator dump` returns `ERROR: null root node returned by UiTestAutomationBridge`, a crashed accessibility service (e.g., Grammarly keyboard) is likely corrupting the bridge. Fix:
```bash
# Check crashed services
adb shell dumpsys accessibility | grep "Crashed services"
# Reset to clear the crash
adb shell settings put secure enabled_accessibility_services ""
# Restore original value later
adb shell settings put secure enabled_accessibility_services "com.grammarly.android.keyboard/..."
```

## Key Event Codes Reference

| Key | Code | Key | Code |
|-----|------|-----|------|
| Enter | 66 | Space | 62 |
| Backspace | 67 | Tab | 61 |
| A-Z | 29-54 | 0-9 | 7-16 |
| Period | 56 | Comma | 55 |
| Minus | 69 | Equals | 70 |
| Apostrophe | 75 | Question | 76 |
| Shift (hold) | 59/60 | | |

## One-Shot Cross-Post Script

Use `post.sh` to post the same content to LinkedIn + Facebook + X in one command:

```bash
# Text-only post
./post.sh "Your post content here"

# Multi-line post
./post.sh -c "Line one

Line two

Line three"

# Post with images
./post.sh "Post text" /tmp/image1.jpg /tmp/image2.jpg

# Multi-line with images
./post.sh -c "Heading

Body paragraph" /tmp/image.jpg
```

The script handles: pushing images to device, opening each app, typing content, attaching images, and publishing. Edit `post.sh` to enable/disable specific platforms.
